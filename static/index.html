What this tool does

This validator checks your product feed against OpenAI's product feed requirements: required vs. recommended fields, dependency rules (e.g., enable_checkout requires enable_search), data types, and formats (prices, dates, URLs, enums, etc.).

How it differs from Google Merchant Center feeds: OpenAI’s schema emphasizes enablement flags (enable_search, enable_checkout), returns/merchant fields, and conversational surfaces. You’ll also see more explicit dependencies (e.g., sale_price requires an effective date range).

  • Max 50,000 rows per run (MVP cap for speed & cost). Split larger feeds into chunks.
  • Supports: JSON arrays, and CSV/TSV (auto delimiter detection).
  • URL validation checks syntax; deep HTTP status checks are skipped to keep validation fast.

      • enable_checkout can be true only when enable_search is true.
      • sale_price requires sale_price_effective_date in format YYYY-MM-DD / YYYY-MM-DD, and must be ≤ price.

      • If you use any of color, size, size_system, gender, you must provide item_group_id.
      • unit_pricing_measure requires base_measure (and vice versa).
